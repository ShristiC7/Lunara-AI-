# Plan 3.3: Node API Integration: Trigger AI, Fetch Insights

## Goal
Connect the Node API to the Python AI service, providing endpoints for users to trigger analysis and retrieve predictions.

## Tasks

<task>
<read_first>
- api/src/services/cycle.service.ts
- api/src/services/symptom.service.ts
</read_first>
<action>
Create `api/src/services/ai.service.ts`.
- `getCyclePrediction(userId)`: Fetches cycle history, calls FastAPI `POST /predict/cycle`, returns result.
- `triggerSymptomAnalysis(userId)`: Fetches symptoms, calls FastAPI `POST /analyze/symptoms`, saves response to `AiInsight` table.
- Implement rate limiting (10 requests/day per user) using Redis/Prisma tally check.
</action>
<acceptance_criteria>
`ai.service.ts` encapsulates all cross-service communication.
</acceptance_criteria>
</task>

<task>
<read_first>
- api/src/controllers/cycle.controller.ts
</read_first>
<action>
Create `api/src/controllers/ai.controller.ts`:
- `getPrediction`: handler for `GET /api/cycles/prediction`.
- `triggerAnalysis`: handler for `POST /api/insights/trigger`. returns 201 when complete.
- `getInsights`: paginated list of insights from DB.
- `getInsightById`: single insight by ID.
</action>
<acceptance_criteria>
Controllers handle the request/response cycle for AI features.
</acceptance_criteria>
</task>

<task>
<read_first>
- api/src/app.ts
</read_first>
<action>
Create `api/src/routes/ai.routes.ts`.
- `GET /insights`: `aiController.getInsights`
- `GET /insights/:id`: `aiController.getInsightById`
- `POST /insights/trigger`: `aiController.triggerAnalysis`
- `GET /cycles/prediction`: `aiController.getPrediction`
Mount in `app.ts` as `/api/insights` and adjust `/api/cycles` if necessary.
</action>
<acceptance_criteria>
Endpoints are reachable and secured by `requireAuth`.
</acceptance_criteria>
</task>
