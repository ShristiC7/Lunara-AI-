# Plan 5.2: Storage (S3) Integration + Report Worker

## Goal
Connect the PDF generator to S3-compatible storage and implement the full background worker to process report requests.

</task>

<task>
<read_first>
- .planning/phases/05-pdf-reports-email-delivery/05-RESEARCH.md
</read_first>
<action>
Create `api/src/utils/s3.ts`.
Implement `StorageService`:
- `upload(key, buffer)`: Uploads to S3.
- `getDownloadUrl(key)`: Generates a 1-hour pre-signed URL.
- Use env vars: `S3_BUCKET`, `S3_REGION`, `S3_ACCESS_KEY`, `S3_SECRET_KEY`, `S3_ENDPOINT`.
</action>
<acceptance_criteria>
Storage service handles secure uploads and pre-signed URL generation.
</acceptance_criteria>
</task>

<task>
<read_first>
- api/src/workers/report.worker.ts
- api/src/utils/pdf.ts
- api/src/utils/s3.ts
</read_first>
<action>
Fully implement `api/src/workers/report.worker.ts` (replacing the stub):
- Extract parameters.
- Gather all user data (cycles, symptoms, insights) from DB.
- Assembly `ReportData` object.
- Call `PdfGenerator.generate()`.
- Call `StorageService.upload()`.
- Record report in `Report` table with S3 key.
- Queue email notification job (to be implemented in 5.3).
</action>
<acceptance_criteria>
Worker successfully processes a user's health history into a stored PDF.
</acceptance_criteria>
</task>
