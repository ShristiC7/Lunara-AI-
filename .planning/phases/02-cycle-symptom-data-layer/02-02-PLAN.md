# Plan 2.2: Symptom Endpoints

## Goal
Build endpoints enabling users to log daily symptoms with data validation, boundaries, and input HTML sanitization, allowing isolated queries attached to cycles.

## Tasks

<task>

- api/package.json
</read_first>
<action>
Install `sanitize-html` and `@types/sanitize-html` (dev). Add them to `package.json` explicitly through a shell install command, ensuring no hang-ups (use standard `npm i sanitize-html`).
</action>
<acceptance_criteria>
`package.json` contains `sanitize-html`.
</acceptance_criteria>
</task>

<task>
<read_first>
- api/prisma/schema.prisma
</read_first>
<action>
Create Zod schema `api/src/schemas/symptom.schema.ts`.
Export `logSymptomSchema` inside `body`: `date` (ISOString), `cycleId` (uuid/optional), `mood` (1-5 limit, optional), `energyLevel` (1-5 limit, optional), `flowIntensity` (0-4 limit, optional), `painLevel` (0-5 limit, optional), `notes` (string max 500 chars, optional).
Export `updateSymptomSchema` partial of the above.
</action>
<acceptance_criteria>
`symptom.schema.ts` exports `logSymptomSchema` strictly validating integers limits.
</acceptance_criteria>
</task>

<task>
<read_first>
- api/src/schemas/symptom.schema.ts
- api/src/utils/errors.ts
</read_first>
<action>
Create `api/src/services/symptom.service.ts`.
Export `SymptomService` class.
- `createSymptom`: takes schema inputs plus `userId`. Apply `sanitizeHtml(notes, { allowedTags: [], allowedAttributes: {} })` if notes provided. Insert to DB using `prisma`. Target matching `userId`.
- `updateSymptom`: similar bounds check, sanitize notes, update if `id` and `userId` match.
- `getSymptomsByCycle`: fetch by `cycleId` && `userId` ordering by `date` desc.
- `getSymptomById`: fetch by `id` && `userId`.
- `deleteSymptom`: execute delete targeting `id` && `userId`.
</action>
<acceptance_criteria>
`symptom.service.ts` uses `sanitize-html` to filter `notes` strings.
`SymptomService` tracks CRUD by `userId` to enforce cross-user boundaries.
</acceptance_criteria>
</task>

Create `api/src/controllers/symptom.controller.ts` mapping the service endpoints.
